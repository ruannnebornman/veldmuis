# Highlights

- Tagged releases now build, sign, and publish the ISO in one GitHub workflow run.
- The website-facing R2 bucket now keeps only `latest.iso`, `latest.iso.sha256`, and `latest.manifest.txt`.
- GitHub Releases now get the built ISO, checksum, and manifest attached as release assets.

# Notes

- This is a prerelease to validate the end-to-end release pipeline before wider install testing.
- Website downloads are expected to resolve through the stable `latest.iso` redirect path.
